Build a full-stack restaurant web app called "DOSA JII" — a South Indian restaurant. Use the MERN stack (MongoDB Atlas, Express.js, React, Node.js) with Tailwind CSS for styling.

=== THEME & BRANDING ===
Tagline: "Karna hai chill toh Dosa Ji se mill"

Color palette (premium light theme — NOT fully dark):
- Page background: warm off-white/cream #FFFDF7
- Section alternates: pure white #FFFFFF and light cream #FFF8EE
- Primary accent: rich amber/gold #E8920A
- Secondary accent: deep saffron #F5A623
- Dark contrast (navbar, footer, hero): #1A1200 (very dark brown-black, not pure black)
- Text primary: #1C1C1C
- Text secondary: #6B6B6B
- Card backgrounds: #FFFFFF with soft golden border #F5C97A at 40% opacity
- Highlight/badge: amber gradient from #F5A623 to #E8920A

Typography:
- Headings: Google Font "Playfair Display" (elegant, premium serif)
- Body & UI: Google Font "Inter"
- Accent/tagline: Google Font "Poppins" semi-bold

Visual style (IMPORTANT):
- NOT a dark website — light creamy backgrounds dominate most pages
- Only the navbar (scrolled state), hero section, and footer use the dark #1A1200 color
- Hero section: dark overlay on a vibrant food background image (use a placeholder gradient from #1A1200 to #3D2200), gold text, tagline in Poppins italic
- Cards: white background, subtle gold/amber border, soft box-shadow (0 2px 16px rgba(232,146,10,0.10)), rounded-xl
- Buttons: primary = solid amber gradient (#F5A623 → #E8920A), white text, rounded-full, subtle hover scale
- Section headings: dark brown #1A1200 with a decorative thin gold underline accent
- Badges (Veg/Non-veg): green pill / red pill, small and clean
- Navbar (default/top): transparent with white logo & links over hero; (scrolled): white background, dark links, gold logo accent, subtle shadow
- Overall feel: think Zomato meets a premium Indian sweets brand — warm, inviting, appetizing, slightly festive

=== USER ROLES ===
Three roles: Customer, Shopkeeper, Admin
- Role-based authentication using JWT tokens stored in localStorage
- Registration: name, email, password, role selection
- Protected routes per role

   - Step indicator: 1. Delivery Details → 2. Payment (gold step bar)

   - Step 2: Order summary + Payment
     * Integrate Razorpay payment gateway (best for India — supports UPI, PhonePe, GPay, cards, net banking, wallets)
     * Backend: POST /api/payment/create-order → calls Razorpay API → returns razorpay_order_id

     * On success: POST /api/payment/verify with razorpay_payment_id, razorpay_order_id, razorpay_signature → verify HMAC on backend → save order to DB with status "Placed"

=== BACKEND API (Node.js + Express) ===
Base URL: /api

Routes: auth (register, login, forgot-password, reset-password), menu (full CRUD + availability toggle), orders (create, my orders, all orders, update status), payment (create Razorpay order, verify signature), users (admin CRUD), coupons (create, list, apply/validate)

=== DATABASE SCHEMAS (MongoDB) ===
- User: name, email, password(bcrypt), role(customer/shopkeeper/admin), phone, address, isActive, createdAt
- MenuItem: name, description, price, category, image(Cloudinary URL), isVeg, isAvailable, isFeatured, createdBy, createdAt
- Order: userId, items[{menuItemId, name, price, qty, image}], deliveryAddress, subtotal, deliveryCharge, gst, couponDiscount, total, razorpayOrderId, razorpayPaymentId, status(Placed/Preparing/OutForDelivery/Delivered/Cancelled), createdAt
- Coupon: code, discountType, value, minOrder, maxUses, usedCount, expiresAt, isActive
